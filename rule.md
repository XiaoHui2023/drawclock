# 规则

## 支持涂鸦

注释、普通形状可以任意涂鸦，但不允许与库器件连线。

## 禁止拉伸

器件图形不允许拉伸，会造成图案变形、连线点偏移等问题。

器件形状应该固定下来并被大众接受，如果对图形大小、间距等不满意，应该重新设计器件库，而不是手动调整。

## 名字独一无二

除 `from` 外，每个器件的 `name` 不重复。

## 输入输出

+ 一个输入只允许连一根线。
+ 一个输出可以连多根线。
+ 单端口器件必须连线
+ 多端口器件输入必须连线，输出可以不连
+ 连线方向应该从一个输出连到另一个输入，不能连反（不画箭头时允许连反）

## from

+ `from` 必须与某张图片中的某个器件同名。
    + 源自一个有单一输出的器件时，等效于连接它的输出。
    + 源自一个没有输出的器件时，等效于连接它的输入。
    + 不允许连接多输出的器件。
+ `from `最终不体现在json中。

## 通用解释器

`drawio转json` 的脚本只做通用的转换器，仅含 `from` 等少量规则。

功能要通用、单纯，只是解释图中的连接关系和器件属性为`json`文件，属性值原样输出，而不做任何逻辑后处理。

这么做的好处是降低维护成本。

假如器件库有增减、修改属性形状等，只需要更新器件库；如果属性值有特殊写法，也应该在之后的脚本处理。

这个脚本将不需要人来维护、不再升级，是一个稳定的通用转换器。

## 重载器件库

器件库更新时，将旧的图片和新的器件库输入脚本，转化为新器件的图片，这样就不用重画图片了。
